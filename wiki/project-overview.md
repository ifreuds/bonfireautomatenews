# Project overview

**The assignment (one sentence):** Build Bonfire an automated newsletter — AI scrapes trusted sources, writes it, publishes it — and turn it into a platform where a small team **approves each issue** before it publishes to the web and pushes to **Lark, Email, Line and WeChat**, each channel getting its own version of the writing and image.

**Who:** Bonfire — a games company / official Tencent Cloud partner in Southeast Asia. Readers are customers and existing partners.

## Where we are (as of 2026-07-02)

| Part | Status |
|---|---|
| Newsletter pipeline (cloud + games) | ✅ Working, live, running daily on a schedule |
| Games newsletter content & format | ✅ Approved by the boss |
| Approval platform (the web app) | 📄 Designed, **not built** |

## Deliverables
1. **A working newsletter** — live site + Lark card, daily. *(done)*
2. **The design document** — `MVP_Approval_Platform_Spec.md`. *(done)*
3. **A clickable unstyled skeleton** of the approval app, for the design team to style. *(remaining)*

Design (company CI) is deliberately the **last leg** — the skeleton is functional and unstyled on purpose.
